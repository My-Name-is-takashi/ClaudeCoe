2-2 IPアドレスとサブネット

🎯 この章で学ぶこと
  • IPv4アドレスとサブネットマスク(CIDR表記)を計算できる
  • ネットワークアドレス・ブロードキャスト・ホスト数を求められる
  • プライベート/グローバルアドレスを判別できる
📺 動画「IPアドレスは住所、サブネットは町内会」(15分)を先にどうぞ。

1. IPアドレスは「町名+番地」でできている

IPv4アドレスは32ビット(8ビット×4区切り)の数値です。前半が「ネットワーク部(町名)」、後半が「ホスト部(番地)」で、その境目を示すのがサブネットマスク(CIDR表記)です。

192.168.10.130/24
└ /24 = 先頭24ビットが町名(ネットワーク部)という意味

2進数で見ると:
192      .168      .10       .130
11000000 10101000 00001010 | 10000010
└──── ネットワーク部(24bit)────┘ └ホスト部(8bit)┘

つまり「192.168.10 町」の「130番地」
CIDRサブネットマスクホスト部使えるホスト数
/24255.255.255.08bit2⁸−2 = 254
/25255.255.255.1287bit2⁷−2 = 126
/26255.255.255.1926bit2⁶−2 = 62
/27255.255.255.2245bit2⁵−2 = 30
/30255.255.255.2522bit2²−2 = 2(機器間接続用)
/16255.255.0.016bit65,534
⚠️ なぜ「−2」? 各サブネットの先頭(ホスト部が全部0)はネットワークアドレス(町名そのもの)、末尾(全部1)はブロードキャストアドレス(町内放送用)で、端末には割り当てられないからです。ここ、テストに出ますよ!

2. 計算の型:「ブロックサイズ」で考える

例題: 192.168.10.130/26 のネットワークアドレスは?

手順① /26 → ホスト部6bit → ブロックサイズ = 2⁶ = 64
手順② 第4オクテットの区間は 0-63 / 64-127 / 128-191 / 192-255
手順③ 130 は 128-191 の区間 → ネットワークアドレスは 192.168.10.128
       ブロードキャストは 192.168.10.191、使える番地は 129〜190(62台)

2進数に毎回変換しなくても、「ブロックサイズの倍数で区切る」と一瞬で解けます。この型をサブネット計算道場で反射になるまで練習しましょう。

3. プライベートIPとグローバルIP

種類範囲用途
プライベート
(社内・家庭内専用)		10.0.0.0/8(10.x.x.x)大規模社内網
172.16.0.0/12(172.16〜31.x.x)中規模。172.32以降は対象外なのが引っかけ頻出
192.168.0.0/16(192.168.x.x)小規模・家庭
グローバル上記以外(おおむね)インターネット上で一意。プロバイダから割当て

社内はプライベートIPを使い、インターネットに出るときにNAT(2-5章)でグローバルIPに変換します。IPv6は「アドレスが枯渇しない新方式(128ビット、2001:db8::1のような表記)」であることをまず押さえれば、現段階ではOKです。

4. 情シスの実務では

  • IPアドレス設計書: 拠点ごと・用途ごと(サーバー/PC/プリンター/ゲスト)にサブネットを割り当てて台帳化
  • 例: ミライ商事 東京 192.168.10.0/24(PC)、192.168.20.0/24(サーバー)、大阪 192.168.30.0/24…
  • 「拠点に60台」なら /26(62台)がぴったり——将来の増加も見込んで1段大きめにするのが現場の知恵
サブネット計算は自転車と同じで、一度体で覚えたら忘れません。道場で満点が出るまで、通勤時間にコツコツどうぞ!
