2-3 TCP/UDPとポート番号

🎯 この章で学ぶこと
  • TCPとUDPの違いと使い分けを説明できる
  • 主要なウェルノウンポートを即答できる
  • 「ポートが開いている」の意味がわかる

1. TCPとUDP 〜書留と速達ハガキ〜

TCPUDP
例えるなら書留郵便(受領確認つき)速達ハガキ(投げっぱなし)
接続3ウェイハンドシェイクで確立してから送るいきなり送る
信頼性届かなければ再送。順序も保証再送なし。速さ優先
主な用途Web(HTTP/HTTPS)、メール、ファイル転送DNS問い合わせ、音声/ビデオ通話、時刻同期
# TCPの3ウェイハンドシェイク(接続のあいさつ)
PC      → サーバー : SYN     「つないでいい?」
サーバー → PC      : SYN/ACK 「いいよ、そっちは?」
PC      → サーバー : ACK     「OK、始めよう」→ データ送受信開始

Web会議の映像がカクつくのは、UDPが「再送より今この瞬間」を選ぶ設計だから。用途によって最適な性質が違うのです。

2. ポート番号 〜同じ住所の中の「窓口番号」〜

IPアドレスが「建物の住所」なら、ポート番号は「何階の何番窓口か」。1台のサーバーがWeb(443)とメール(587)とSSH(22)を同時に受け付けられるのはポートのおかげです。

ポートプロトコル用途
80 / 443HTTP / HTTPSWeb(443が暗号化)
22SSHサーバーの遠隔操作(Part 3-9)
25 / 587SMTP / Submissionメール送信(587が投稿用)
110 / 143POP3 / IMAPメール受信(995/993がSSL版)
53DNS名前解決(2-4章)
67/68DHCPIP自動配布(2-5章)
3389RDPWindowsリモートデスクトップ
445SMBWindowsファイル共有
123 / 161 / 514NTP / SNMP / Syslog時刻同期 / 監視 / ログ転送

「ポートが開いている」とは、そのポートで待ち受けているプログラム(サービス)が存在するということ。FWの許可設定(Part 6)も、この番号単位で行います。

3. 自分のPCで確かめる

# Windows: 通信中の接続と待ち受けポートを見る
netstat -an | findstr LISTEN

# Linux(Part 3の環境ができたら): ssコマンドが現代の定番
ss -tlnp    # TCPの待ち受け(Listen)を表示。nginxを入れると:80が現れる
💪 やってみよう(10分)
  1. netstat -an を実行し、「:443」への接続(ESTABLISHED)を探す — 今ブラウザが開いているHTTPS接続の実体です
  2. ポート番号かるたで10問中8問以上を目指す
ポート番号は情シスの「九九」。FW設定でもトラブル調査でも毎日出てきます。かるたで遊んでいるうちに勝手に覚えちゃいますよ。
📌 まとめ
  • TCP=確実(再送・順序保証)、UDP=速さ優先。用途で使い分け
  • ポート番号=窓口番号。「開いている」=待ち受けサービスがいる
  • 443/22/53/25/3389/445 あたりは即答レベルに
⚡ ポート番号かるたで特訓する 📝 章末クイズに挑戦する(6問)
