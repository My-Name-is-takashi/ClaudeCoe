Part 2 ネットワーク基礎と実践

2-5 DHCP・NAT・ルーティング

🎯 この章で学ぶこと
  • DHCPの4ステップ(DORA)とリース運用を説明できる
  • NAT/NAPTの仕組みとポートフォワードを説明できる
  • デフォルトゲートウェイと経路の考え方を体感する

1. DHCP 〜IPアドレスの自動配布係〜

# DORA: PCがネットワークに参加するときの4ステップ
① Discover  PC「誰かIPください!」(ブロードキャスト)
② Offer     DHCPサーバー「192.168.10.55 はどう?」
③ Request   PC「それください!」
④ Ack       サーバー「どうぞ。リース期間は8時間ね」
  • リース期間: 貸出期限。切れる前に自動更新される
  • 枯渇障害: 配れるIPが尽きると新しい端末がつながらない。「午後から来た人だけWi-Fiに入れない」の定番原因
  • 予約(固定割当て): プリンターやサーバーはMACアドレスに紐づけて毎回同じIPを配る、または静的設定にする

2. NAT/NAPT 〜社内の顔とインターネットの顔〜

社内のプライベートIPのままではインターネットと通信できません。ルーターが送信元を会社のグローバルIPに書き換えて代表発信します。これがNAT。さらにポート番号も使って「社内の誰の通信か」を管理し、1つのグローバルIPを全員で共有するのがNAPT(IPマスカレード)です。

社内(変換前)→インターネット側(変換後)
192.168.10.5:50001→203.0.113.1:60001
192.168.10.8:50002→203.0.113.1:60002

ポートフォワードはこの逆向きの穴あけ。「グローバルIPの443番に来た通信は社内の192.168.20.10へ渡す」のように、外部公開に使います(安易な公開はPart 6で学ぶリスクと表裏一体)。

3. ルーティング 〜「知らない宛先はぜんぶ親分へ」〜

  • PCは通信のたびに考える: 宛先は同じ町内(サブネット)?
  • 同じ町内 → スイッチ経由で直接届ける(ARPでMACを調べて)
  • 違う町 → デフォルトゲートウェイ(ルーター)に丸投げ
  • ルーターはルーティングテーブル(行き先表)を見て次のルーターへ… の連鎖で世界中に届く
# 拠点間のスタティックルートの例(ミライ商事)
宛先 192.168.30.0/24(大阪) → ネクストホップ 10.0.0.2(VPNルーター)
宛先 0.0.0.0/0(その他ぜんぶ) → 203.0.113.254(インターネット出口)
# 0.0.0.0/0 が「デフォルトルート」。上から具体的な経路が優先される

4. ラボ: 自分の端末の通信経路を調べる(45分)

# ① 自分の設定を確認(Windows)
ipconfig /all
   → IPv4アドレス / サブネットマスク / デフォルトゲートウェイ / DHCPサーバー / DNSサーバー を書き出す

# ② ゲートウェイのMACアドレスをARPテーブルで確認
arp -a | findstr <ゲートウェイのIP>

# ③ 外部までの経路を見る(各行が経由ルーター)
tracert -d 8.8.8.8
   → 1行目が自宅/社内ルーター、2行目以降がプロバイダ網

# ④ DHCPのリースを更新してみる
ipconfig /release && ipconfig /renew   # 一瞬ネットが切れます(自宅で!)

✅ 確認: 「自分のPCがインターネットに出るまでの登場人物(IP・GW・DNS・NAT)」を紙1枚に描ければクリア。

「IPは来てる?」「GWにpingは通る?」「DNSは引ける?」——この3点セットが言えるようになったら、Part 2の折り返し地点は突破です!
📌 まとめ
  • DHCPはDORAの4ステップ。枯渇とリースは障害対応の頻出ネタ
  • NAPTで社内全員が1つのグローバルIPを共有。逆向きの穴がポートフォワード
  • 知らない宛先はデフォルトゲートウェイへ。経路は tracert で見える
