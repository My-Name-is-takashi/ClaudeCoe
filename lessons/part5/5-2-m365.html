5-2 Microsoft 365管理

  • Entra ID(旧Azure AD)とオンプレADの関係を説明できる

1. 管理センターの地図

管理センター担当情シスの日常業務
Microsoft 365 管理センターユーザー・ライセンス・請求の玄関口入退社対応、ライセンス割当
Entra ID(旧Azure AD)認証・グループ・条件付きアクセスSSO設定、MFA、サインインログ調査
Exchange 管理センターメール共有メールボックス、転送、迷惑メール対策
SharePoint / OneDriveファイル共有外部共有ポリシー、サイト管理
Teams 管理センターチャット・会議チーム作成ポリシー、外部アクセス
Defender / Purviewセキュリティ・情報保護脅威対策、監査ログ検索(超重要)

2. Entra IDとオンプレADの関係

オンプレAD
mirai.local(Part 4)⇄ Entra Connectで同期 ⇄Entra ID
クラウドの認証基盤
ハイブリッド構成: 社内はAD、クラウド(M365等)はEntra ID。同じID・パスワードで両方使える。
  • Entra IDは「クラウド版の社員名簿+鍵束」。M365のログインはすべてここが処理
  • ハイブリッドでは「どちらが正(マスター)か」を意識——同期元(通常オンプレAD)で編集しないと元に戻ってしまう
  • 新しい会社ではオンプレADなしの「Entra IDオンリー」構成も増加中

4. 現場で多い管理タスク

  • 共有メールボックス: info@ など。追加ライセンス不要(50GBまで)で作れる——問い合わせ窓口の定番
  • 退職者のメール: 共有メールボックス化してライセンス回収、が定石(4-5章のチェックリストと連動)
  • 外部共有: SharePoint/OneDriveの外部共有は「既定で誰でもリンク可」になっていないか必ず点検
  • 監査ログ: 「誰がこのファイルを消した?」はPurviewの監査ログ検索で追える。有効化されているか最初に確認
💪 ラボ: M365管理演習(150分・試用テナント)
  1. M365開発者プログラムまたは試用版でテナントを作成(会社のテナントでは絶対に演習しない)
  2. ユーザー3名を作成し、ライセンスを割当て。1名は「ライセンスなし」で挙動の違いを確認
  3. 共有メールボックス info@ を作成し、メンバーからの送受信を確認
  4. SharePointの外部共有設定を「既存ゲストのみ」に変更してみる
  5. Entra IDでサインインログを開き、自分のログイン履歴(場所・端末)を読む
  6. 監査ログ検索を有効化し、ファイル削除操作を検索してみる
M365の管理画面は頻繁に模様替えします。「画面の場所」ではなく「どの管理センターの担当か」で覚えるのが長持ちのコツですよ。
📌 まとめ
  • 入口はM365管理センター、認証の心臓はEntra ID
  • ハイブリッドは「マスターはどちらか」を常に意識
  • 共有メールボックス・外部共有設定・監査ログが実務の3大頻出
