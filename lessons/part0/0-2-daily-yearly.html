0-2 情シスの1日・1年 〜リアルな業務カレンダー〜

🎯 この章で学ぶこと
  • 日次・月次・年次の定常業務と突発業務の違いがわかる
  • 情シスの年間イベント(繁忙期)を把握する
📺 動画「情シスの1日密着(平常日/障害日)」(12分)とあわせてどうぞ。

1. 情シスのある1日(平常運転の日)

時刻業務種別
9:00監視アラート・メール・チケット確認(夜間に何もなかったか)日次定常
9:30問い合わせ対応(パスワードリセット、会議室の映像トラブル)突発
11:00来週入社する中途社員のPCキッティング定常(入社対応)
13:00ベンダーと保守契約更新の打ち合わせ計画業務
15:00サーバーのセキュリティパッチ適用計画づくり計画業務
16:30「共有フォルダーが開けない」問い合わせ→権限設定ミスを修正突発
17:30チケット記録・明日の準備日次定常

ポイントは、計画業務と突発対応が常に混ざること。だから優先度判断(Part 7)と記録の習慣(チケット駆動)が大切になります。

2. 障害の日は、こう変わる

朝9時に「全社でファイルサーバーにつながらない」となれば、その日の予定はすべて吹き飛びます。影響範囲の把握 → 社内アナウンス → 原因切り分け → 復旧 → 報告書。この流れはPart 7-6「障害対応総合訓練」で疑似体験します。

障害の日は心臓がドキドキします。でも「型」を知っていれば、手が止まらなくなります。この講座で型をたくさん持ち帰ってくださいね。

3. 情シスの1年カレンダー

時期イベントやること
1〜3月新年度準備・予算確定新入社員のPC・アカウント準備(4月入社分)、来期予算の最終調整
4月入社ラッシュ ★繁忙期キッティング・アカウント発行・IT研修・問い合わせ急増
5〜6月落ち着く時期棚卸し、たまった改善タスク、資格学習のチャンス
7〜9月監査・防災内部監査/ISMS審査対応、BCP訓練、電源設備点検
10月下期開始・人事異動異動に伴う権限変更・座席/ネットワーク変更
11〜12月来期予算策定 ★重要更新対象の洗い出し、見積取得、予算申請。年末年始の停止計画
通年毎月Windows Update(パッチチューズデー対応)、入退社対応、ライセンス棚卸し
💪 やってみよう(5分)
  1. 「4月1日に新入社員が20人入社する」と想像してください
  2. 3月中に準備すべきことを、思いつく限り書き出してみましょう
  3. 書けたら下の模範解答例と見比べてください
📄 模範解答例(クリックで表示)
  • PC20台の調達(納期は1〜2ヶ月前に発注)・キッティング
  • ADアカウント・メールアドレス・各SaaSアカウントの発行(命名規則の確認)
  • ライセンス数の確認・追加購入
  • 座席・ネットワーク(有線/無線)・電話の準備
  • 入社日のITオリエンテーション資料・初期パスワードの配布方法
  • ヘルプデスク体制の増強(4月は問い合わせが倍増)
📌 まとめ
  • 情シスの毎日は「計画業務+突発対応」のミックス。記録と優先度判断が命
  • 4月(入社)と11〜12月(予算)が2大繁忙期。準備は前倒しが鉄則
