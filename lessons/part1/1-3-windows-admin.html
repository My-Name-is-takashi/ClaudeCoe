Part 1 コンピュータとOSの基礎

1-3 Windowsクライアント管理入門(ラボ)

🎯 このラボのゴール(所要 約90分)
  • ユーザーアカウント・UAC・Windows Update・イベントログを実際に操作する
  • 管理ツールを「実行コマンド」で呼び出せるようになる
前提: Windows 10/11 のPC(会社PCではなく私物や検証機で!)。管理者権限が必要です。
⚠️ ラボの鉄則: 会社の本番PCでは絶対に実施しないこと。自分のPCで行う場合も、手順にある変更は最後に元へ戻します。

STEP 1: 管理ツールを呼び出す(10分)

# Win+R(ファイル名を指定して実行)から。情シスの「合言葉」たちです
winver      → Windowsのバージョン確認
msinfo32    → システム情報(型番・BIOS・メモリ)
eventvwr    → イベントビューアー
control     → コントロールパネル
compmgmt.msc→ コンピューターの管理(ユーザー・ディスク・サービス)
taskschd.msc→ タスクスケジューラ

✅ 確認: 6つすべて開けたらSTEP 1完了。この呼び出し方は電話サポートでも「Win+Rで◯◯と打ってください」とそのまま使えます。

STEP 2: ローカルユーザーと権限(20分)

  1. compmgmt.msc →「ローカルユーザーとグループ」→「ユーザー」
  2. 右クリック→「新しいユーザー」: 名前 labuser、パスワードを設定して作成
  3. 作成した labuser をダブルクリック→「所属するグループ」を確認(既定は Users=標準ユーザー)
  4. いったんサインアウトし、labuser でサインイン→ソフトのインストールを試みると管理者パスワードを要求されることを確認(これが UAC)
  5. 元のユーザーに戻る
💡 なぜ大事? 会社では「一般社員=標準ユーザー、情シス=管理者」が大原則。マルウェアに感染しても、標準ユーザーなら被害を大幅に抑えられます(Part 6 で再登場)。

STEP 3: Windows Update の制御(15分)

  1. 「設定」→「Windows Update」→「更新の履歴」で最近当たった更新を確認
  2. 「詳細オプション」→「更新の一時停止」を1週間に設定してみる → すぐ解除する
  3. 用語確認: 毎月第2火曜(米国時間)の定例更新=パッチチューズデー。翌水曜は問い合わせが増えがち

STEP 4: イベントビューアーでログを読む(25分)

  1. eventvwr →「Windowsログ」→「システム」
  2. 「レベル」列で並べ替え、エラーと警告をいくつか開いて説明文を読む
  3. 「現在のログをフィルター」で「レベル: エラー」「期間: 過去7日」に絞り込む
  4. ソース Kernel-Power ID:41 があれば、それは「正常終了せず電源断」の痕跡(突然のシャットダウン調査の定番)

✅ 確認: 「過去7日のエラーだけを一覧にする」操作が一人でできればSTEP 4完了。

STEP 5: タスクスケジューラ(15分)

  1. taskschd.msc →「タスクスケジューラライブラリ」を眺め、既存タスクがたくさん動いていることを確認
  2. 「基本タスクの作成」→ 名前 lab-notepad → 毎日 → 時刻は5分後 → プログラム notepad.exe → 完了
  3. 5分待ってメモ帳が自動起動することを確認 → タスクを削除して後片付け

うまくいかないとき

  • 「ローカルユーザーとグループ」が無い → Home エディションです。「設定」→「アカウント」→「他のユーザー」で代用できます(1-2章のエディション差の実体験!)
  • UACのダイアログが出ない → UAC設定が最低になっていないか control→ユーザーアカウントで確認
📌 まとめ
  • 管理ツールは実行コマンド(winver/msinfo32/eventvwr…)で瞬時に呼べる
  • 標準ユーザー+UACが社内セキュリティの土台
  • イベントログは「過去7日のエラーに絞る」から始めるのが読み方の型
