1. 要件書(これだけを頼りに構築してください)

📋 ミライ商事 社内お知らせサイト構築 要件書
  • OS: Ubuntu Server 24.04 LTS(mirai-lab01 を使用可)
  • 構成: nginx + PHP-FPM + MariaDB(いわゆる LEMP)
  • アプリ: 疎通確認用の info.php と、DB接続確認ページ db.php を配置
  • 公開: HTTPSのみ(自己署名可)。HTTPは443へリダイレクト
  • 運用: サイト資材とDBを毎日3:00にバックアップ(7世代)
  • 納品物: ①動くサイト ②構築手順書 ③完成チェックリストの自己検収結果

2. 進め方 〜ヒントは3段階〜

まず自力で構築順序を書き出してから着手してください。詰まったら下のヒントを1段ずつ開きます(いきなり全部開けたら負けです!)。

💡 ヒント1: 構築順序(クリックで表示)
  1. スナップショット取得(戻れる場所を作る)
  2. apt update/upgrade → nginx / php-fpm / mariadb-server を導入
  3. MariaDB初期設定(mysql_secure_installation)→ アプリ用DB・ユーザー作成
  4. nginxにPHP-FPM連携のserver定義(location ~ \.php$ を fastcgi_pass へ)
  5. info.php / db.php 配置 → HTTP で疎通確認
  6. 自己署名証明書 → 443化+リダイレクト
  7. バックアップスクリプト+cron(3-8章の再利用)
💡 ヒント2: 詰まりやすい箇所
  • php-fpm のソケットパスは /run/php/php8.3-fpm.sock(バージョンで変わる。ls /run/php/ で確認)
  • 502 Bad Gateway → nginxは動いているがPHP-FPMに届いていない(status確認+ソケットパス)
  • db.php の接続エラー → DBユーザーの権限(GRANT)とパスワード、ホスト指定(localhost)を確認
  • DBバックアップは mysqldump を tar と組み合わせる
💡 ヒント3: 検証コマンド集 
systemctl status nginx php8.3-fpm mariadb   # 3役の生存確認
sudo nginx -t && sudo systemctl reload nginx
curl -kI https://localhost/                 # 200が返るか(-kは自己署名許容)
curl -k  https://localhost/info.php | head  # PHPが処理されているか
sudo -u www-data ls /var/www/site           # 権限は足りているか(3-6章!)
journalctl -u nginx --since "10 min ago"    # 直近のエラー確認

3. 完成チェックリスト(自己検収・20項目抜粋)

#確認項目合格基準
1https://でトップページ表示200 OK(自己署名の警告は許容)
2http://アクセス301で443へリダイレクト
3info.phpPHP情報が表示される(=FPM連携OK)
4db.php「DB接続OK」が表示される
5OS再起動後全サービスが自動起動し1〜4が再現(enable漏れ検出)
6バックアップcron登録済み+手動実行でtar.gzとdumpが生成される
7世代管理8世代目を置くと最古が消える(find -mtime テスト)
8権限資材はwww-dataが読める最小権限(777が1つもない)
9設定検証nginx -t がクリーン
10手順書コマンドを写せば第三者が再現できる粒度
