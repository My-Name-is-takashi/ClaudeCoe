3-8 シェルスクリプト入門とcron

🎯 この章で学ぶこと
  • バックアップスクリプトを書ける(変数・if・for の最小文法)
  • cron で定期実行を仕込める
  • 事故らないスクリプトの作法(ドライラン・ログ)を身につける
※ 本講座の方針どおり「プログラミング」には踏み込みません。定型運用の自動化に必要な最小限だけ扱います。

1. 最小文法 〜これだけで運用は回る〜

#!/bin/bash                      # シバン: このファイルはbashで動かす宣言
TARGET="/srv/keiri-share"        # 変数(=の前後にスペースを入れない!)
DEST="/backup"
DATE=$(date +%Y%m%d)             # コマンドの結果を変数に入れる

if [ ! -d "$DEST" ]; then        # if: DESTが無ければ作る(変数は"で囲む)
  mkdir -p "$DEST"
fi

for dir in eigyo keiri soumu; do # for: 繰り返し
  echo "processing $dir ..."
done

tar czf "$DEST/backup-$DATE.tar.gz" "$TARGET"
echo "backup done: $DATE" >> /var/log/mybackup.log
  • 終了コード: 直前のコマンドの成否は $?(0=成功)。コマンドA && コマンドB(成功したら)/ ||(失敗したら)で分岐できる
  • 実行方法: chmod +x backup.sh して ./backup.sh
⚠️ 事故防止の3作法 ①破壊系(rm等)を含むスクリプトは、まず該当行を echo に置き換えてドライラン ②変数は必ず "$VAR" と引用符で囲む(空だった時の rm -rf / 事故を防ぐ) ③実行結果をログに追記して、あとから追えるようにする。

2. cron 〜Linux版タスクスケジューラ〜

crontab -e     # 自分の定期ジョブを編集(初回はエディタ選択→viでもnanoでも)
crontab -l     # 登録内容の確認

# 書式: 分 時 日 月 曜日 コマンド
30 2 * * *   /home/mirai/backup.sh            # 毎日 2:30
0  9 * * 1   /home/mirai/weekly-report.sh     # 毎週月曜 9:00
*/10 * * * * /home/mirai/check.sh             # 10分ごと
0  0 1 * *   /home/mirai/monthly.sh           # 毎月1日 0:00

読み方のコツ: 左から「分・時・日・月・曜日」、*は「毎」。「30 2 * * *」=「毎日、2時30分」。1-3章のWindowsタスクスケジューラと同じ役割です。

💪 ラボ: はじめての運用自動化(120分)
  1. ~/bin/backup.sh を作成: /srv/keiri-share(3-6章で作成)を日付付きtar.gzで /backup に退避する(上のコードを土台に)
  2. 手動実行して /backup にファイルができること、ログが追記されることを確認
  3. 世代管理を追加: find /backup -name "backup-*.tar.gz" -mtime +7 -delete を「まず -delete なしで」試してから組み込む(7世代より古い分を削除)
  4. crontab -e で「毎日2:30」に登録。テスト用に「3分後」の時刻でも登録し、実際に動くことを確認したらテスト行を削除
  5. 仕上げ: 実行に失敗したらログに「FAILED」と書くよう || echo "FAILED $DATE" >> ログ を追加
自動化のご褒美は「自分が休んでいる間も仕事が回る」こと。ただし動いたことを確認する仕組み(ログ・監視)までがセット。「動いているはず」は情シス禁句です!
📌 まとめ
  • 変数・if・for・終了コードだけで運用スクリプトは書ける
  • cronは「分 時 日 月 曜日」。まず読めること、次に書けること
  • ドライラン・引用符・ログの3作法で事故を防ぐ
