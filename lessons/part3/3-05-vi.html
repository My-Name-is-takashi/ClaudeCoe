1. なぜ今さらvi?

サーバーの設定ファイルを編集するとき、GUIのエディタは使えません。SSHでつないだ黒い画面の中で編集する必要があるからです。そして、viはどんなLinuxサーバーにも必ず入っています。障害対応で初めて触るサーバーでも、viさえ使えれば設定を直せる——だから情シスの必修スキルなのです。

最初はみんな「vi、閉じ込められた!助けて!」ってなります。わたしもなりました。でもモードの仕組みが分かれば、3日で怖くなくなりますよ!

2. viの心臓部:モードを理解する

viが難しく感じる原因はただひとつ、「モード」があることです。普通のエディタは「開いたら即、文字が打てる」のに対し、viは命令を出すモードと文字を打つモードが分かれています。

挿入モード文字を入力する
ノーマルモード起動直後はココ/移動・削除・コピー
コマンドラインモード保存・終了・置換
← i a o で入る
Esc で戻る → ← : / で入る
Esc で戻る →

覚えることは実は2つだけです。

  1. 迷子になったら、まず Esc(何度押してもOK。必ずノーマルモードに戻れる)
  2. 全部やめて逃げたいときは Esc → :q!(保存せずに終了)
⚠️ ここ、テストに出ますよ!
:q! は「保存せずに強制終了」。編集が消えてもいいときの緊急脱出です。保存して終了は :wq。この2つを取り違えると、30分の編集作業が消えます。

💪 やってみよう①(5分)
  1. 練習ファイルを作る: cp /etc/hosts ~/hosts.practice
  2. vi ~/hosts.practice で開く
  3. G で最終行へ → o で行を追加 → 192.168.10.5 testserver と入力
  4. Esc → :wq で保存終了 → cat ~/hosts.practice で確認

4. 実務の主役:検索と置換

設定ファイルの編集で最も使うのが検索と置換です。たとえばサーバーのIPアドレスが変わったとき:

# ノーマルモードで入力(コロンから始まる)
:%s/192.168.1.10/192.168.1.20/g
#  │  │           │            └ g: 行内のすべての出現を対象
#  │  │           └ 新しい文字列
#  │  └ 古い文字列
#  └ %: ファイル全体の行を対象

置換前に /192.168.1.10 で検索して n で件数感覚を掴んでから置換すると、事故が減ります。「検索で確認 → 置換で実行」が安全な型です。

5. サーバー設定ファイルを安全に編集する作法

viが使えることと、本番サーバーの設定を安全に編集できることは別のスキルです。情シスの現場では必ずこの手順を踏みます。

# ① 編集前に必ずバックアップ(日付入り)
sudo cp /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak.20260706

# ② sudoを付けてviで編集
sudo vi /etc/nginx/nginx.conf

# ③ 構文チェック(nginxの場合)— reloadの前に必ず!
sudo nginx -t

# ④ 問題なければ反映
sudo systemctl reload nginx

# ⑤ 動作確認(戻せる状態のまま検証)
curl -I http://localhost
⚠️ sudoを付け忘れてviで開いてしまったら?
保存時に「readonly」エラーが出ても慌てないでください。:w /tmp/nginx.conf.tmp で一時退避すれば編集内容は失われません。退避後に sudo cp で反映しましょう。:q! でやり直すと編集が全部消えます。
💪 やってみよう②(10分)
  1. 3-2章で作ったVMにSSH接続する
  2. sudo cp /etc/ssh/sshd_config ~/sshd_config.practice で練習用コピーを作る
  3. vi ~/sshd_config.practice で開き、/PermitRootLogin で検索
  4. その行を yy → p で複製し、複製した行を PermitRootLogin no に書き換える
  5. u を2回押してすべて取り消し、Ctrl+r で1つだけやり直してみる
  6. :q! で保存せずに終了する(練習ファイルなので気軽に!)

6. まとめ

  • viはノーマル・挿入・コマンドラインの3モード。迷ったら Esc、脱出は :q!
  • 実務の9割は15個のキーで回る。ドリルで反射になるまで練習する
  • 本番編集は「バックアップ → 編集 → 構文チェック → reload → 動作確認」の型を必ず守る
