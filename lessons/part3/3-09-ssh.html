Part 3 Linuxサーバー入門〜実践

3-9 SSHとファイル転送

🎯 この章で学ぶこと
  • 公開鍵認証の仕組みを図で説明でき、鍵ペアを作って設定できる
  • scp / rsync で安全にファイルを転送できる
  • sshd_config の要塞化(rootログイン禁止・パスワード認証禁止)ができる

1. 公開鍵認証 〜南京錠と鍵〜

公開鍵 = 南京錠
サーバーに置く(見られてOK) 秘密鍵 = その鍵
手元に置く(絶対に渡さない)
サーバーは南京錠で施錠した“お題”を送り、正しい鍵を持つ人だけが解錠できる。パスワードと違い、盗聴・総当たりに強い。 
# ① 手元PCで鍵ペア生成(Ed25519が現代の標準)
ssh-keygen -t ed25519 -C "mirai@lab"
   → ~/.ssh/id_ed25519(秘密鍵)と id_ed25519.pub(公開鍵)ができる

# ② 公開鍵をサーバーへ登録
ssh-copy-id mirai@192.168.56.101
   (Windowsでssh-copy-idが無い場合)
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh mirai@192.168.56.101 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

# ③ パスワードを聞かれずログインできれば成功
ssh mirai@192.168.56.101
⚠️ パーミッション要件: サーバー側 ~/.ssh は 700、authorized_keys は 600 でないと鍵認証は動きません(3-6章の知識がここで直結!)。「鍵を入れたのに入れない」の原因第1位です。

2. sshd_config の要塞化 〜締め出されない手順で〜

sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak.20260706   # ① バックアップ
sudo vi /etc/ssh/sshd_config                                      # ② viで編集

PermitRootLogin no          # rootの直ログイン禁止
PasswordAuthentication no   # パスワード認証禁止(鍵のみ)

sudo sshd -t                # ③ 構文チェック(nginx -t のSSH版)
sudo systemctl reload ssh   # ④ 反映

鉄の掟: 今つながっているSSHセッションを閉じずに、新しいターミナルで接続テスト。失敗しても既存セッションから設定を戻せます。これを怠って自分を締め出すのが「SSH三大事故」の筆頭です。

3. ファイル転送: scp と rsync

scp report.txt mirai@192.168.56.101:/tmp/          # 手元→サーバー
scp mirai@192.168.56.101:/var/log/syslog ./         # サーバー→手元
scp -r ./website mirai@192.168.56.101:/srv/         # ディレクトリごと

rsync -av --delete ./website/ mirai@192.168.56.101:/srv/website/
# rsync=差分同期。2回目以降は変更分だけ。--deleteは消えたファイルも同期(要注意オプション)

単発はscp、繰り返し・大量はrsync。~/.ssh/config に接続先を書いておくと ssh lab01 だけで入れて世界が変わります:

# ~/.ssh/config
Host lab01
  HostName 192.168.56.101
  User mirai
  IdentityFile ~/.ssh/id_ed25519
💪 ラボ: SSH要塞化(90分)
  1. 鍵ペア生成 → 公開鍵登録 → パスワードなしログイン確認
  2. ~/.ssh/config を書いて ssh lab01 で入れるようにする
  3. sshd_config を上の手順で要塞化(既存セッション維持+別ターミナルで検証を必ず実践)
  4. パスワード認証で入れなくなったことを確認: ssh -o PubkeyAuthentication=no lab01 → 拒否されればOK
  5. scpで任意のファイルを往復させ、rsyncで2回目が一瞬で終わることを体感する
秘密鍵はパスワード以上の宝物。チャットで送るのも、共有PCに置くのも厳禁です。「南京錠(公開鍵)は配ってよし、鍵(秘密鍵)は肌身離さず」ですよ!
📌 まとめ
  • 公開鍵=南京錠(サーバーへ)、秘密鍵=手元厳守。~/.sshは700、鍵は600
  • 設定変更は「バックアップ→vi→sshd -t→reload→別ターミナル検証」
  • 単発scp・繰り返しrsync。~/.ssh/configで接続を名前化
