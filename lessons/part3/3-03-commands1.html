3-3 基本コマンドI(ファイル操作・パイプ・リダイレクト)

🎯 この章で学ぶこと
  • ファイル操作の基本コマンドを迷わず使える
  • 絶対パスと相対パスを使い分けられる
  • パイプとリダイレクトでコマンドを組み合わせられる

1. まずは移動と閲覧

pwd                 # 今いる場所(print working directory)
ls                  # ファイル一覧。詳細は ls -l、隠しファイル込みは ls -la
cd /var/log         # 移動(絶対パス)。cd だけでホームへ、cd .. で1つ上へ
cat motd            # 短いファイルを表示
less syslog         # 長いファイルを1画面ずつ(q で終了、/文字列 で検索)
パスの種類例意味
絶対パス/etc/nginx/nginx.confルート(/)からの完全な住所。手順書ではこちらを使う
相対パス../log/syslog今いる場所から見た道順(.=ここ、..=1つ上)
ホーム~/memo.txt~=自分のホーム(/home/mirai)

2. 作る・コピーする・消す

mkdir -p work/2026/07     # -p で深い階層を一気に作成
touch memo.txt            # 空ファイル作成
cp memo.txt memo.bak      # コピー。ディレクトリごとは cp -r
mv memo.bak work/         # 移動 兼 リネーム
rm work/memo.bak          # 削除(ゴミ箱はない!)
⚠️ rm に「元に戻す」はありません。特に rm -rf は破壊力最大。プロの作法は、rmの前に同じパス指定で ls を打って対象を目視確認すること。ここ、テストに出ますよ!

3. パイプとリダイレクト 〜コマンドの合体技〜

# | (パイプ): 左の出力を右の入力へ渡す
ls -l /etc | less              # 長い一覧を1画面ずつ
history | grep ssh             # 過去に打ったsshコマンドを探す

# > と >> (リダイレクト): 出力をファイルへ
ls -l /etc > etc-list.txt      # 上書き保存
date >> work-log.txt           # 追記(ログはこちら)

# 2> : エラーだけ別ファイルへ
find / -name "*.conf" 2> /dev/null   # エラーを捨てて結果だけ見る定番

「小さな道具をパイプでつなぐ」のがLinux流。この感覚がPart 3後半のログ調査で効いてきます。

4. 効率3種の神器

  • Tab補完: 途中まで打ってTab。ファイル名の打ち間違いが消滅する
  • 履歴: ↑↓で過去のコマンド、Ctrl+R で履歴検索
  • ワイルドカード: *.log(任意文字列)、file?.txt(任意1文字)
💪 ラボ: ファイル操作ノック(前半・90分)
  1. mirai-lab01 にSSH接続し、~/dojo ディレクトリを作って移動
  2. 部署ディレクトリ eigyo keiri soumu を一気に作る(ヒント: mkdirは複数引数OK)
  3. 各部署に meibo.txt を touch で作成し、ls */meibo.txt で確認
  4. /etc/hostname を ~/dojo にコピーし、hostname.bak にリネーム
