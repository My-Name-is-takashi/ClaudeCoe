3-1 Linuxとは・ディストリビューションの選び方

🎯 この章で学ぶこと
  • Linuxが社内のどこで動いているかを挙げられる
  • ディストリビューションの系統(Debian系/RHEL系)と選定観点を説明できる
  • 「なぜサーバーはCUIなのか」に答えられる
📺 動画「世界はLinuxでできている」(10分)を先にどうぞ。

1. 世界はLinuxでできている

Webサーバーの大多数、社内のNAS、ルーターやFWの中身、スマホ(Android)、クラウドの仮想マシン——情シスが触る「箱」の中身はかなりの割合でLinuxです。ミライ商事でも社内ポータル(nginx)と監視サーバーがLinuxで動いています。

  • カーネル: Linuxの本体(OSの芯)。ハードウェアを管理する
  • ディストリビューション(ディストリ): カーネル+コマンド群+パッケージ管理などを束ねた「配布パッケージ」。UbuntuやRed Hatはこれ
  • ライセンス: GPL等のオープンソース。無償で使えるが、商用サポートを買うという選択肢がある(ここが企業選定の肝)

2. ディストリビューション系統図

Debian系→Ubuntu / Ubuntu Server
本講座の標準。情報が豊富
RHEL系→RHEL(商用サポート付き)
+互換のAlmaLinux/Rocky Linux
2大系統。コマンドの基本は共通だが、パッケージ管理(apt / dnf)と細部の流儀が異なる。
観点Ubuntu(Debian系)RHEL系
サポートLTS版は5年(無償)商用契約で長期・手厚い
パッケージ管理apt(.deb)dnf(.rpm)
採用場面クラウド・Web・学習基幹系・ベンダー製品の動作要件
⚠️ LTSとサポート期限(EOL): LTS(長期サポート版)以外は9ヶ月程度で更新が止まります。サーバーには必ずLTS(例: Ubuntu 24.04 LTS)を選び、EOL管理は情シスの責任と心得ましょう(Part 6-4に続く伏線です)。

3. なぜサーバーはCUI(黒い画面)なのか

  • 軽い: GUIを描かない分、資源をサービスに回せる
  • 遠隔・自動化と相性抜群: SSH越しに文字だけで完結。手順はそのままスクリプトになる
  • 再現性: 「このコマンドを打つ」は誰がやっても同じ。GUIの「ここをクリック」より手順書が正確になる
黒い画面は「不親切」じゃなくて「合理的」なんです。慣れるとGUIより速くて、しかも作業ログが全部残る。3-3章からたっぷり触りますよ!
📌 まとめ
  • サーバー・NAS・ネットワーク機器の中身はLinuxだらけ
  • 系統はDebian系(Ubuntu)とRHEL系。サーバーはLTS一択、EOL管理は情シスの責任
  • CUIは軽さ・遠隔・自動化・再現性のための合理的な選択
