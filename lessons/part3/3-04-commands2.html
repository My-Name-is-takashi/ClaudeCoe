Part 3 Linuxサーバー入門〜実践

3-4 基本コマンドII(検索・圧縮・プロセス)

🎯 この章で学ぶこと
  • grep と find で「探す」を制する(業務最頻出)
  • tar でアーカイブを扱える
  • ps・top・kill でプロセスを制御できる

1. grep 〜中身から探す〜

grep error /var/log/syslog          # errorを含む行を表示
grep -i error syslog                # -i 大文字小文字を無視
grep -n "192.168.10.5" *.conf       # -n 行番号つき。設定から特定IPを探す定番
grep -r "server_name" /etc/nginx/   # -r ディレクトリごと再帰検索
grep -v "^#" nginx.conf             # -v 「含まない行」= コメントを除いて読む神業
ps aux | grep nginx                 # パイプ合わせ技: プロセスの中から探す

正規表現は初歩の4つだけまず覚えます: ^(行頭)/ $(行末)/ .(任意の1文字)/ *(直前の繰り返し)。grep "^Port" sshd_config のように「行頭がPortの行」を狙い撃ちできます。

2. find 〜名前・日付・サイズで探す〜

find /etc -name "*.conf"            # 名前で探す
find /var/log -mtime -7             # 7日以内に更新されたファイル
find / -size +100M 2>/dev/null      # 100MB超の大物(ディスクフル調査の主砲)
find ~/dojo -name "*.txt" -delete   # 見つけて削除(先に -delete なしで確認!)

grep=中身で探す、find=属性で探す。この分業を覚えれば「あのファイルどこ?」に困りません。

3. tar 〜固める・展開する〜

tar czf logs-20260706.tar.gz /var/log/nginx/   # c=作成 z=gzip f=ファイル名
tar tzf logs-20260706.tar.gz                   # t=中身を見る(展開前の確認作法)
tar xzf logs-20260706.tar.gz -C /tmp/restore/  # x=展開 -C=展開先指定

「czf(作る)・tzf(覗く)・xzf(開ける)」の3点セットで暗記。バックアップ(3-8章)の土台です。

4. プロセスとリソース

ps aux                    # 全プロセス一覧(USER/PID/%CPU/%MEMを見る)
top                       # リアルタイム監視(q で終了、M でメモリ順)
kill 1234                 # PID 1234へ終了要求(TERM=お願い)
kill -9 1234              # 強制終了(KILL=問答無用。最後の手段)
df -h                     # ディスク使用量(100%はないか?)
du -sh /var/log/*         # どのディレクトリが太っているか
free -h                   # メモリの空き
⚠️ kill -9 をいきなり撃たない: TERM(既定)はプロセスに後片付けの猶予を与えます。-9は後片付けなしの即死で、データ破損の恐れ。まず kill、効かないときだけ -9。
💪 ラボ: 実務シナリオノック(後半・90分)
  1. 「設定ファイル群から特定IPを探せ」: grep -rn "127.0.0.1" /etc/ 2>/dev/null | head を実行し、出力の意味を説明する
  2. 「先週分のログを固めて退避せよ」: /var/log で7日以内更新の .log を find で列挙 → tar czf で ~/backup-$(date +%Y%m%d).tar.gz に固める → tzf で中身確認
  3. 「暴走プロセスを止めろ」: 別ターミナルで yes > /dev/null & を実行 → top でCPU100%の犯人を見つけ、PIDを調べて kill → topで消えたことを確認
  4. 「ディスクの太った場所を探せ」: du -sh /var/* 2>/dev/null | sort -h | tail -5 でワースト5を出す
今日の4つ(grep・find・tar・ps/kill)は、障害対応の現場で毎回登場するレギュラーメンバー。3-12章のログ調査で「あ、これ3-4でやったやつ!」ってなりますよ。
