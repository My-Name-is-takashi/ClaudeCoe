3-7 パッケージ管理とsystemd

🎯 この章で学ぶこと
  • apt でパッケージの検索・導入・更新・削除ができる
  • systemctl でサービスの起動・停止・自動起動を制御できる
  • journalctl でサービスのログを追える

1. apt 〜ソフトの「公式アプリストア」〜

sudo apt update            # カタログ(パッケージ一覧)を最新化 ※まだ何も入らない
sudo apt upgrade           # 入っているソフトを一括更新(セキュリティ運用の基本)
apt search nginx           # 探す
apt show nginx             # 詳細(バージョン・説明)を見る
sudo apt install nginx     # インストール(依存関係も自動解決)
sudo apt remove nginx      # 削除(設定は残す)。完全削除は purge
⚠️ update と upgrade の違い: update=カタログの更新、upgrade=ソフト本体の更新。「update したのに直らない」はこの取り違えが原因のことも。セットで sudo apt update && sudo apt upgrade と覚えましょう。

リポジトリ(配布元)から署名検証つきで取得する仕組みなので、野良サイトからのダウンロードより安全。これが「Linuxにウイルス対策ソフトの概念が薄い」一因です(無敵という意味ではありません!)。

2. systemd 〜サービスの元締め〜

systemctl status nginx     # 状態確認(active (running)? failed?)
sudo systemctl start nginx    # 起動
sudo systemctl stop nginx     # 停止
sudo systemctl restart nginx  # 再起動(接続は一瞬切れる)
sudo systemctl reload nginx   # 設定再読込(無停止。対応サービスのみ)
sudo systemctl enable nginx   # OS起動時の自動起動をON
sudo systemctl disable nginx  # 自動起動OFF
混同しがちペア違い
start と enablestart=「今」動かす / enable=「次回起動時から」自動で動く。両方やって初めて「今も再起動後も動く」
restart と reloadrestart=全部入れ直し(瞬断あり)/ reload=設定だけ読み直し(無停止)。本番ではまずreloadを検討

3. journalctl 〜サービスの日記帳〜

journalctl -u nginx            # nginxのログだけ表示
journalctl -u nginx -f         # リアルタイム追尾(tail -f 相当)
journalctl -u nginx --since "1 hour ago"
journalctl -xe                 # 直近のログ+解説つき(起動失敗調査の定番)

起動失敗の調査手順は「status → journalctl -xe」の2連コンボ。statusで概要(いつ・何番の終了コードで死んだか)、journalctlで詳細(なぜ)を見ます。

💪 ラボ: サービス管理演習(75分)
  1. スナップショット取得 → sudo apt update && sudo apt install nginx
  2. systemctl status nginx で active (running) を確認 → curl http://localhost でWelcomeページのHTMLが返ることを確認
  3. sudo systemctl stop nginx → curlが失敗することを確認 → start で復活
  4. systemctl is-enabled nginx で自動起動を確認 → sudo reboot → 再接続して「再起動後も動いている」ことを確認
  5. journalctl -u nginx で起動・停止の履歴が時系列で残っていることを確認
  6. おまけ: /etc/nginx/nginx.conf の1行目にわざと hoge と書いて sudo systemctl restart nginx → 失敗を status と journalctl -xe で調査 → viで直して復旧(3-5章の実戦!)
手順6の「わざと壊して直す」、ちょっとドキドキしましたか?本番で初体験するより、ラボで100回転んでおくほうが、ずっと安上がりです!
📌 まとめ
  • apt: update(カタログ)→ upgrade(本体)。導入はinstall一発
  • systemctl: start/enable は別物。本番の設定反映はまず reload
  • 起動失敗は「status → journalctl -xe」の2連コンボで調べる
