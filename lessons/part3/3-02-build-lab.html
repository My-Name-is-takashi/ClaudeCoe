Part 3 Linuxサーバー入門〜実践

3-2 学習環境構築(VirtualBox+Ubuntu Server)

🎯 このラボのゴール(所要 約120分)
  • 仮想マシンに Ubuntu Server 24.04 LTS をインストールする
  • スナップショットで「壊しても戻せる」自分専用サーバーを持つ
  • ホストPCからSSH接続できる状態にする(Part 3全ラボの母艦になります)
📺 動画「仮想化ってなに?」(8分)を先にどうぞ。

STEP 0: 仮想化のイメージを掴む

仮想マシン=「PCの中のPC」
ホストOS(あなたのWindows/Mac)→VirtualBox(ハイパーバイザー)→ゲストOS(Ubuntu Server)
壊しても消しても本体は無傷。スナップショット=セーブポイント。

STEP 1: VirtualBox とISOの入手(15分)

  1. VirtualBox 公式サイトから自分のOS用インストーラーを取得して導入
  2. Ubuntu 公式サイトから Ubuntu Server 24.04 LTS のISOを取得(Desktop版ではなくServer版!)
  3. 可能ならISOのチェックサム(SHA256)を照合(ダウンロード破損・改ざり検知の作法)

STEP 2: VM作成(15分)

設定推奨値意味
名前mirai-lab01以後の教材はこの名前で呼びます
CPU2コアホストの物理コアの半分以下に
メモリ2048MB(余裕があれば4096)ホストが8GBなら2048に留める
ディスク25GB(可変サイズ)可変なら実使用分しか消費しない
ネットワークアダプター1: NAT
アダプター2: ホストオンリー		NAT=外へ出る用 / ホストオンリー=ホストからSSHする用の2枚構成

STEP 3: Ubuntu Server インストール(30分)

  1. VMを起動しISOから boot → 言語は English 推奨(サーバーはエラーメッセージ検索性が命)
  2. ネットワーク: 2枚のNICが見えることを確認(DHCPのままでOK)
  3. ストレージ: 「Use an entire disk」でOK(LVMはチェックのまま)
  4. プロファイル: ユーザー名 mirai / サーバー名 mirai-lab01 / 強いパスワード
  5. 「Install OpenSSH server」に必ずチェック(これを忘れると後でひと手間)
  6. インストール完了 → 再起動 → ログインプロンプトが出れば成功!

STEP 4: SSH接続とスナップショット(20分)

# VMのコンソールでIPを確認(ホストオンリー側: 192.168.56.x のはず)
ip a

# ホストPC(Windowsのターミナル/PowerShell)から接続
ssh mirai@192.168.56.101   # ←表示されたIPに読み替え
# 初回は「fingerprintを信頼するか」→ yes。パスワードを入れてログイン

# つながったら、中を少し見てみる
hostname && whoami && ip a && df -h
  1. SSHでログインできたら、いったん exit
  2. VirtualBoxで スナップショット作成: 名前「clean-install」— これが今後の「セーブポイント」
⚠️ 以後の全ラボの約束: 大きな変更の前にスナップショット。壊れたら「clean-install」に戻して再挑戦。壊すことを恐れない環境こそ、この章の成果物です。

うまくいかないとき(FAQ)

  • VMが起動しない/64bitが選べない → BIOSで仮想化支援(VT-x/AMD-V)が無効。有効化する。WindowsではHyper-V/メモリ整合性との競合も確認
  • ホストオンリーのIPが付かない → VirtualBoxの「ホストネットワークマネージャー」でDHCP有効を確認、VM内で ip a の2枚目を確認
  • SSHで接続拒否 → OpenSSH選択漏れ。VMコンソールで sudo apt install openssh-server
🔀 代替手順: メモリ8GB未満のPC → WSL2(Windows)や UTM(Apple Silicon Mac)でUbuntuを導入してもPart 3の大半のラボは実施できます(スナップショット相当はWSLのexport/importで代用)。
