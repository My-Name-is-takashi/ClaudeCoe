Part 7 IT運用とヘルプデスク

7-3 IT資産管理とライセンス管理

🎯 この章で学ぶこと
  • ハード・ソフト・ライセンスの台帳を設計できる
  • 棚卸しプロセスを回せる
  • 廃棄時のデータ消去を正しく行える

1. 台帳の設計 〜3つの台帳と1つの原則〜

台帳必須列更新タイミング
ハードウェア台帳管理番号 / 型番・シリアル / 利用者 / 設置場所 / 購入日・保証期限 / EOL / 状態(利用中・保管・廃棄)キッティング時・入退社・移動時(1-4章)
ソフトウェア台帳製品名・バージョン / インストール先 / EOL導入・更新時(資産管理ツールで自動収集が理想)
ライセンス台帳製品 / ライセンス種別 / 保有数 vs 利用数 / 契約更新日 / 証書の保管場所購入・割当・回収時(5-6章のSaaS台帳と統合可)

原則はただ一つ: 「現物・台帳・実際の利用者が常に一致している」。ズレは事件(紛失・シャドーIT・ライセンス違反)の兆候です。

2. ライセンスの型と監査リスク

  • デバイス/ユーザー/同時使用/コア課金/サブスク——製品ごとに数え方が違う。「保有数<利用数」が違反状態
  • ベンダーからのライセンス監査(真正性確認)は実際に来ます。違反時は正規料金+ペナルティの購入で、数百万円規模になることも
  • 逆に「保有数>利用数」は無駄なコスト。棚卸しはリスクと費用の両方を削る一石二鳥の仕事です

3. 棚卸しの回し方(半期に1回)

① 資産管理ツール/AD/MDMから「機械が知っている一覧」を出す
② 台帳と突合 → 差分リストを作る(台帳にあるのに見えない端末=最優先調査)
③ 現物確認(利用者にセルフチェック依頼+抜き取り実査)
④ 差分の原因を潰す(返却漏れ・記載漏れ・紛失)→ 台帳更新
⑤ 結果を記録(ISMS審査・内部統制の証跡になる: 6-7章)

4. 廃棄 〜データ消去までが資産管理〜

⚠️ 「初期化したから大丈夫」ではない: 通常のフォーマットではデータは復元可能です。廃棄時は①暗号化消去(BitLocker等の鍵破棄)②消去ソフトによる上書き ③物理破壊のいずれかを行い、業者委託時はデータ消去証明書を必ず受領・保管します。HDD横流しによる情報流出は実際に起きた重大事件です。

📄 模範解答
  • NB-0088: 退職者に貸与中のまま=回収漏れ(情報漏えいリスク・即回収)
  • NB-0089: Windows 10はEOL済み=パッチ無しで稼働中(6-4章のEOL管理違反)
  • 山本さんのMac: 管理番号なし・購入日不明=台帳外資産(シャドーIT疑い。経緯確認と登録)
  • NB-0090: 「保管中」なのに所在不明=紛失の疑い。データ暗号化の有無を確認し、規程に沿って報告
  • 全体: 2019年購入で6年超=更新計画(8-2章)の対象
台帳仕事は地味ですが、「あのPC誰が使ってる?」に3秒で答えられる情シスは、監査でも障害でも人事でも頼られます。地味こそ武器です。
📌 まとめ
  • 台帳は3つ(ハード・ソフト・ライセンス)+EOL列。現物と一致してこそ台帳
  • ライセンスは「保有vs利用」の差分管理。監査は実際に来る
  • 廃棄はデータ消去+証明書まで。フォーマットは消去ではない
