9-2 卒業試験

🎓 卒業試験の構成
  • 知識試験: 全パート横断30問(80%で合格)
  • 実技試験: Linuxサーバー復旧課題(下記)
  • 両方の合格+9-3の卒業制作提出で「一人前情シス認定」です
📝 知識試験(30問)を開始する

実技試験: 壊れたWebサーバーを復旧せよ

📋 課題(制限時間: 90分)

mirai-lab01 のスナップショットを取得したうえで、以下の「破壊スクリプト」を実行し、自力で完全復旧してください。

# 破壊スクリプト(3つの故障を注入します。実行前にスナップショット!)
sudo systemctl stop nginx && sudo systemctl disable nginx
sudo sed -i 's/}$/#}/' /etc/nginx/sites-enabled/default 2>/dev/null || \
  sudo sed -i '0,/}/s//#}/' /etc/nginx/nginx.conf
sudo chmod 000 /var/www/html
echo "破壊完了。ここから90分で復旧してください。"
合格条件(すべて達成)
  • □ curl -I http://localhost が 200 を返す
  • □ OS再起動後もnginxが自動起動して200を返す
  • □ nginx -t がクリーン
  • □ ファイル権限が適切(777を使っていない)
  • □ 報告書: 3つの故障の内容・発見方法(使ったコマンドとログ)・対処・再発防止を A4 1枚に
💡 ヒント(どうしても詰まったら)
  • 故障は「サービス停止+自動起動OFF」「設定ファイルの構文破壊」「ドキュメントルートの権限剥奪」の3つ
  • 調査の型: systemctl status → nginx -t(行番号を読む!)→ ls -l /var/www → curl
  • 権限は「なぜ644/755が適切か」を報告書で説明できること

不合格だったら

  • 知識試験: 間違えた問題の解説に書かれた章へ戻って復習 → 3日以上あけて再挑戦(問題はシャッフルされます)
  • 実技: スナップショットに戻して何度でも。2回目で30分を切れたら、実務レベルの型が身についた証拠です
