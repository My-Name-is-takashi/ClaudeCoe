Part 4 Windows ServerとActive Directory

4-1 Windows Serverの役割と管理ツール

🎯 この章で学ぶこと
  • Windows Serverが社内で担う役割(AD/DNS/DHCP/ファイル/印刷)を説明できる
  • 管理ツール群(サーバーマネージャー・MMC・RSAT)の位置づけがわかる
  • CAL(ライセンス)の概念を費用感として説明できる
📺 動画「Windows Serverは会社の“総務部長”」(10分)とあわせてどうぞ。

1. Windows Serverは「役割」の集合体

Windows Serverは、必要な機能を「役割(Role)」として追加して使うOSです。ミライ商事の社内で担っているのはこの5役:

役割何をする?止まると…
AD DS(Active Directory)社員名簿+認証の元締め(4-2章)全員がログオン不能。全社障害級
DNSサーバー社内の名前解決(ADと一心同体)サーバー名でアクセス不能、ADも不調に
DHCPサーバー社内へのIP自動配布(2-5章の社内版)新規接続端末がIPを取れない
ファイルサーバー部署共有フォルダー(4-4章)「共有が開けない」問い合わせ殺到
印刷サーバープリンターの一元配布全社で印刷不能

2. 管理ツール図鑑

  • サーバーマネージャー: 役割の追加・状態確認の玄関口
  • MMC+スナップイン: 「Active Directory ユーザーとコンピューター(dsa.msc)」「グループポリシー管理(gpmc.msc)」など、◯◯.msc は全部MMCの仲間
  • RSAT: 自分のWindows 11にインストールする管理ツール集。サーバーに毎回リモートデスクトップしなくても手元から管理できる——実務の基本形
  • PowerShell: 一括処理の味方(例: 退職者100人の無効化)。本講座では「存在と用途」を知ればOK

3. GUIとServer Core

Windows ServerにはGUIあり/なし(Server Core)の2形態があります。Coreは軽く攻撃面も小さいためDC(ドメインコントローラー)向きですが、本講座の検証はGUIありで進めます(学習効率優先)。

4. ライセンスの基本(CAL)

⚠️ サーバー本体のライセンスだけでは使えません
Windows Serverは本体ライセンス(コア数課金)に加えて、アクセスする人or端末の数だけCAL(Client Access License)が必要です。「社員300名がADにログオンする」=300ユーザーCAL。監査で指摘される定番ポイントなので、台帳管理(Part 7-3)とセットで覚えましょう。
  • ユーザーCAL(人単位)とデバイスCAL(端末単位)——多くの会社は1人複数端末なのでユーザーCALが得
  • Entra ID(旧Azure AD)とのハイブリッド構成は5-2章で扱います
Windows Serverは「会社の総務部長」——名簿(AD)を持ち、鍵(認証)を管理し、備品(ファイル・プリンター)を貸し出す。この比喩でPart 4全体が見通せますよ。
📌 まとめ
  • Windows Serverは役割の集合体。AD DS+DNSが心臓部
  • 管理はRSATで手元から。◯◯.mscはMMCスナップイン
  • CAL(ユーザー/デバイス)を忘れると監査で痛い目に
