Part 4 Windows ServerとActive Directory

4-3 グループポリシー(GPO)

🎯 この章で学ぶこと
  • GPOで端末設定を一括統制できる
  • 適用順序(LSDOU)と継承を理解し、「当たらない」を切り分けられる
  • gpupdate / gpresult で検証できる

1. GPOは「社内ルールの自動配布装置」

「全PCの画面ロックを10分に」「USBメモリ書き込み禁止」——300台に手作業したら日が暮れます。GPO(グループポリシーオブジェクト)は、設定のかたまりをOUにリンクして、対象のユーザー/PCに自動適用する仕組みです。

構成対象例
コンピューターの構成PC(誰がログオンしても適用)画面ロック強制、USB制限、Windows Update設定
ユーザーの構成人(どのPCでも適用)壁紙、ドライブマッピング、スタートメニュー制御

2. 適用順序 LSDOU 〜あとから当たったものが勝つ〜

L(Local)     ローカルポリシー         ↓ 弱い
S(Site)      サイト
D(Domain)    ドメイン                 例: パスワードポリシー
OU(OU)       OU(入れ子なら親→子の順)  ↓ 強い(最後に適用=優先)
  • 子OUは親の設定を継承する。同じ項目がぶつかったら「後から(=より近いOU)」が勝つ
  • 例外の道具: 「継承のブロック」「強制(Enforced)」「セキュリティフィルター(特定グループのみ適用)」——便利だが乱用すると追えなくなる
⚠️ パスワードポリシーはドメイン直下のみ: 部署ごとに違うパスワード要件をOUのGPOで作っても効きません(既定ではドメイン単位)。試験にも実務にも出る有名な罠です。

3. 検証コマンド2点セット

gpupdate /force     # クライアントで即時適用(通常は90分±30分ごとに自動適用)
gpresult /r         # このPC/ユーザーに「実際に当たっているGPO」を表示

「GPOが当たらない」切り分けは: ①gpresultで対象GPOが見えるか → ②リンク先OUにそのPC/ユーザーがいるか → ③フィルターで除外されていないか → ④「コンピューター構成」をユーザーOUに当てていないか(対象違い)。

💪 ラボ: GPO演習(120分)
  1. グループポリシー管理(gpmc.msc)で「東京」OUに新規GPO PC-Lock-Policy を作成
  2. コンピューターの構成→「10分でスクリーンセーバーロック」を設定し、PC用OUにリンク
  3. クライアントで gpupdate /force → 設定が変わったこと、gpresult /r に表示されることを確認
  4. ユーザーの構成で「壁紙を会社ロゴに固定」→ 別ユーザーでログオンしても壁紙が変わることを確認
  5. ドライブマッピング: 「Gドライブ=\\dc01\share」をユーザー構成の基本設定で配布(4-4章の伏線)
  6. 実験: 同じ項目を親OUと子OUで違う値にして、子が勝つことをgpresultで確認
GPOのコツは「欲張って1つの巨大GPOにしない」こと。目的ごとに小さく作って名前で分かるようにすると、1年後の自分が泣かずに済みますよ。
📌 まとめ
  • GPO=設定のかたまり×OUへのリンク。PC向けとユーザー向けは別物
  • 適用はLSDOU、近いOUが勝つ。パスワードポリシーはドメイン単位
  • 検証はgpupdate /force → gpresult /r の2点セット
