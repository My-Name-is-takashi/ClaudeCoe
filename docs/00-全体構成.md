# 情シス養成講座「ゼロから一人前の情シスへ」全体構成書

## 1. 教材の目的とゴール

**目的**: IT初心者(PCの基本操作はできるが、業務としてのITは未経験)が、
中小〜中堅企業で **一人で情シス業務を回せるレベル(=一人前)** に到達すること。

**ゴール像(修了時にできること)**:

| 領域 | 修了時の到達レベル |
|---|---|
| PC・OS | キッティング、トラブル一次切り分け、MDMでの端末管理ができる |
| ネットワーク | IP設計の読み書き、DNS/DHCP/VLANの理解、障害の切り分けができる |
| Linuxサーバー | コマンド操作、viでの設定編集、nginx等の設定の読解・変更、環境構築ができる |
| Windows/AD | ADのユーザー・GPO運用、ファイルサーバーの権限設計ができる |
| クラウド/SaaS | M365/Google Workspaceの管理、SSO/MFAの設計、AWSの基本操作ができる |
| セキュリティ | 脅威の理解、脆弱性・パッチ運用、インシデント初動対応ができる |
| 運用・実務 | ヘルプデスク、資産管理、監視、バックアップ、ベンダー管理、稟議が書ける |

**対象外(スコープ外)**: プログラミング(アプリ開発)、データベース設計、
機械学習などの専門開発領域。ただしシェルスクリプト程度の「運用自動化」は含む。

**想定学習時間**: 約 200 時間(1日1時間 × 約6〜7ヶ月 / 集中なら3ヶ月)

## 2. 媒体構成(飽きさせない7つの学習形式)

単一媒体の連続を避け、**1章の中で必ず2種類以上の媒体を組み合わせる**ことを設計原則とする。

| # | 媒体 | 用途 | 分量の目安 | 備考 |
|---|---|---|---|---|
| M1 | **解説動画** | 新しい概念の導入。図解アニメ+ナレーション | 1本 5〜15分 | ナレーターは明るくかわいい女性ボイス(仕様は媒体制作ガイド参照) |
| M2 | **HTML読み物教材** | 動画の詳細補足、リファレンス、図解記事 | 1本 10〜20分で読める分量 | 図・表・コードブロック多用。ダークモード対応 |
| M3 | **インタラクティブクイズ** | 章末の理解度チェック。即時採点+解説表示 | 1章 5〜20問 | 4択・複数選択・穴埋め・並べ替えの4形式 |
| M4 | **ハンズオンラボ** | 実機演習。手順書+確認チェックリスト | 1本 30〜90分 | VirtualBox/WSL2/クラウド無料枠で完結する構成 |
| M5 | **ドリル** | 反復訓練(viキー操作、サブネット計算、コマンド)| 1回 5分 | ゲーム感覚のスコア付き。スキマ時間用 |
| M6 | **ケーススタディ(分岐シナリオ)** | 「あなたならどうする?」選択肢分岐型の疑似体験 | 1本 15〜30分 | 障害対応・ヘルプデスク・インシデント対応で使用 |
| M7 | **修了テスト** | パート末の総合試験。合格ライン80% | 1回 20〜50問 | 不合格章の復習リンクを自動提示 |

### 媒体の使い分けルール

- **概念の導入は必ず動画(M1)から**。文字だけで新概念を説明しない。
- **手を動かせるものは必ずラボ(M4)化**。読んだだけで終わらせない。
- **暗記・反射が必要なもの(viのキー、サブネット計算)はドリル(M5)化**。
- **判断力が必要なもの(障害対応・インシデント)はケーススタディ(M6)化**。
- クイズ(M3)は章末に必ず置き、**間違えた問題には該当教材へのリンク**を付ける。

### 動画ナレーター設定(全動画共通)

- キャラクター: 案内役「**いろは**」(新人に寄り添う先輩、20代女性)
- 声質: **明るくかわいい女性ボイス**。高めのトーン、ゆっくりはっきり、語尾がやわらかい
- 口調: 「〜ですよ」「〜してみましょう!」「ここ、テストに出ます!」など親しみやすく
- 専門用語の直後に必ず一言の言い換えフォローを入れる(例: 「DNS、つまり“インターネットの電話帳”ですね」)
- 詳細な音声仕様・台本テンプレートは `docs/01-媒体制作ガイド.md` を参照

## 3. カリキュラム全体マップ(10パート・70章)

```
レベル0 ─ Part 0  オリエンテーション:情シスという仕事      (4章 /  約6時間)
レベル1 ─ Part 1  コンピュータとOSの基礎                  (6章 / 約15時間)
       └ Part 2  ネットワーク基礎と実践                   (9章 / 約25時間)
レベル2 ─ Part 3  Linuxサーバー入門〜実践 ★最重量パート    (13章 / 約40時間)
       └ Part 4  Windows ServerとActive Directory        (6章 / 約20時間)
レベル3 ─ Part 5  クラウドとSaaS管理                      (7章 / 約25時間)
       └ Part 6  情報セキュリティ                        (8章 / 約25時間)
レベル4 ─ Part 7  IT運用とヘルプデスク                    (7章 / 約20時間)
       └ Part 8  情シス実務スキル                        (7章 / 約15時間)
卒業   ─ Part 9  総合演習と卒業試験                       (3章 / 約10時間)
```

- **推奨順序は Part 0 → 9 の直列**だが、Part 3 と Part 4、Part 7 と Part 8 は並行学習可。
- 各パートの詳細な章構成は `docs/curriculum/part0.md`〜`part9.md` を参照。

### レベル定義(社内スキル認定にそのまま使える粒度)

| レベル | 呼称 | 到達基準 |
|---|---|---|
| 0 | 見習い | 情シスの業務範囲と自分の学習計画を説明できる |
| 1 | アシスタント | PCセットアップと社内ネットワークの一次切り分けができる |
| 2 | オペレーター | サーバー(Linux/Windows)の日常運用・設定変更ができる |
| 3 | 担当者 | クラウド・SaaS・セキュリティを含む社内IT全体を運用できる |
| 4 | 一人前 | 運用設計・ベンダー管理・予算まで含めて一人で情シスを回せる |

## 5. 学習体験の設計(飽きさせない仕組み)

1. **1セッション30分設計**: 「動画1本+クイズ1セット」または「ラボ1ステップ」が30分で完結。
2. **進捗の可視化**: 学習ポータル(`portal/index.html`)でチェックリスト・進捗率・レベルバッジを表示。
3. **媒体のリズム**: 「見る(動画)→読む(HTML)→解く(クイズ)→触る(ラボ)」のサイクルを章単位で回す。
4. **スキマ時間対応**: ドリル(M5)はスマホで5分でできる設計。
5. **物語性**: ケーススタディは架空の会社「株式会社ミライ商事(従業員300名)」を舞台に統一。
   Part 0 で入社し、Part 9 で「一人前」として障害対応を指揮するストーリーライン。
6. **ご褒美設計**: パート修了ごとにバッジ発行。卒業試験合格で「一人前情シス認定証」(印刷用HTML)。
