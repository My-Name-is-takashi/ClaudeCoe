# Part 6 情報セキュリティ(全8章 / 約25時間 / レベル3)

## パート概要

情シスの最重要責務。攻撃手法を知り、防御を設計し、インシデントに対応する。
知識偏重を避け、**標的型メール訓練・インシデント対応シミュレーション**などの
体験型教材を中心に据える。

**到達目標**: 自社のセキュリティ対策状況を評価でき、インシデント発生時の初動対応を
指揮でき、経営層に対策の必要性を説明できる。

## 各章詳細

### 6-1 セキュリティの基礎と攻撃手法

- **学習ゴール**: CIA(機密性・完全性・可用性)で対策を整理でき、主要な攻撃手法の流れを説明できる
- **教材構成**
  - 動画①「攻撃者は誰で、何が目的か」(12分): ランサムウェア被害の実例ベースドラマ(架空化)
  - 動画②「攻撃の教科書:侵入から暗号化まで」(13分): サイバーキルチェーンを防御側視点で図解
  - クイズ: 12問
- **内容トピック**: CIA / 脅威・脆弱性・リスクの関係 / ランサムウェア / フィッシング / サプライチェーン攻撃 / 内部不正 / IPA「情報セキュリティ10大脅威」の読み方(毎年更新される前提の教材設計)

### 6-2 マルウェアとエンドポイント防御

- **学習ゴール**: AVとEDRの違いを説明でき、感染疑い端末の初動(隔離)ができる
- **教材構成**
  - 動画「アンチウイルスはもう古い?EDRの世界」(10分)
  - HTML「エンドポイント防御スタック図解」: OS標準機能(Defender)→EPP→EDR→MDRの階層
  - ミニ演習: 「感染疑いの連絡を受けた。最初の3手は?」(順序並べ替え問題)
- **内容トピック**: マルウェアの種類 / シグネチャと振る舞い検知 / EDRのアラート読解(サンプル画面)/ 隔離手順 / 「駆除できました」で終わらせない再発防止

### 6-3 ネットワークセキュリティとゼロトラスト

- **学習ゴール**: 境界型防御の構成要素(FW/UTM/IDS/IPS/プロキシ)とゼロトラストの考え方を対比して説明できる
- **教材構成**
  - HTML「境界型防御 完全図解」: ミライ商事のネットワーク図にFW・UTM・プロキシを配置していく段階図解
  - HTML「ゼロトラスト入門」: 「社内=安全」が崩れた理由、SASE/ZTNAの概観
  - クイズ: 12問(FWルールの読解問題: 「このACLで通る通信はどれ?」)
- **内容トピック**: FWルール設計(暗黙のdeny)/ DMZ / IDS/IPS / Webフィルタリング / ゼロトラストの原則 / 移行は一夜にしてならず(現実的なロードマップ)

### 6-4 脆弱性管理とパッチ運用

- **学習ゴール**: CVE/CVSSを読めるようになり、パッチ適用の優先順位付けと計画を立てられる
- **教材構成**
  - ラボ「脆弱性トリアージ演習」(90分): 架空の脆弱性スキャン結果(20件)を配布し、CVSSと自社環境への該当性から対応優先順位を付けて計画表を作る → 模範解答と突き合わせ
  - HTML「CVE/CVSSの読み方」: 実在のCVE公開情報の構造解説
- **内容トピック**: 脆弱性情報源(JVN/NVD/ベンダーアドバイザリ)/ CVSSの構成 / 「即パッチ」できない現実との折り合い(緩和策・計画メンテ)/ EOL管理

### 6-5 標的型攻撃メール対応訓練

- **学習ゴール**: 不審メールを見抜くチェックポイントを体得し、開封後の正しい行動を反射でとれる
- **教材構成**
  - **インタラクティブ演習「受信トレイ・シミュレーター」**: 20通のメール(正常13通+攻撃7通)を実際の受信トレイ風UIで判定していくゲーム形式。判定後に着眼点(送信元ドメイン・リンク先・文面の圧力)を解説
  - 動画「開いてしまった!そのあと1分でやること」(6分)
  - HTML「社内向け注意喚起メールのテンプレート」(情シス側の発信業務も体験)
- **内容トピック**: なりすましの見抜き方(表示名とアドレス、リンクのホバー確認)/ SPF/DKIM/DMARCの概要 / 報告文化の作り方(開封者を責めない)

### 6-6 インシデント対応

- **学習ゴール**: 検知→初動→封じ込め→復旧→報告 の流れを指揮でき、報告書を書ける
- **教材構成**
  - ケーススタディ(分岐型・大型シナリオ2本):
    1. 「ランサムウェア感染:金曜18時の悪夢」(60分): 隔離範囲・バックアップ確認・経営報告・警察/IPA相談・身代金の論点まで、時系列で判断を迫る本格シナリオ
    2. 「誤送信:全顧客リストを社外へ」(30分): 個人情報漏えい時の報告義務(個情法)と顧客対応
  - HTML「インシデント対応計画テンプレート」+「初動チェックリスト(壁貼り用1枚)」
- **内容トピック**: CSIRTと役割分担 / 証拠保全(勝手に再起動しない)/ 報告のタイムライン(個人情報保護委員会への報告義務)/ 事後の再発防止レポート

### 6-7 法令・規格とセキュリティポリシー

- **学習ゴール**: 個人情報保護法の実務ポイント、ISMS(ISO27001)とPマークの違いを説明でき、社内ポリシー文書の階層を理解する
- **教材構成**
  - HTML「情シスのための法令・規格ガイド」: 個人情報保護法 / 不正アクセス禁止法 / 電子帳簿保存法(関与ポイントのみ)/ ISMS・Pマーク比較
  - HTML「ポリシー文書3階層(基本方針・対策基準・実施手順)」+サンプル文書
  - クイズ: 12問
- **内容トピック**: 個人データの安全管理措置 / 漏えい時の報告義務 / 監査対応の実際 / 「規程はあるが守られない」問題への処方箋

### 6-8 Part 6 修了テスト

- 35問 / 80%合格(インシデント初動の順序問題を必ず含む)
