# Part 5 クラウドとSaaS管理(全7章 / 約25時間 / レベル3)

## パート概要

現代の情シス業務の主戦場。Microsoft 365 / Google Workspace の管理、
SSO・MFAによるID管理、AWSの基礎までを扱う。
「オンプレの知識(Part 2〜4)がクラウドでどう置き換わるか」の対応付けを常に示す。

**到達目標**: M365/Google Workspaceのテナント管理、SSO/MFAの設計説明、
AWS無料枠での基本構築ができ、SaaSの選定・管理プロセスを回せる。

## 各章詳細

### 5-1 クラウドの基礎(IaaS/PaaS/SaaS)

- **学習ゴール**: サービスモデルと責任共有モデルを図で説明でき、「クラウド移行」の判断軸を持つ
- **教材構成**
  - 動画「ピザで学ぶ IaaS/PaaS/SaaS」(10分): 定番の比喩を丁寧なアニメで
  - HTML「責任共有モデル早見表」: どこまでがクラウド事業者/自社の責任か、モデル別比較
  - クイズ: 10問
- **内容トピック**: オンプレとの費用構造比較(CAPEX/OPEX)/ リージョンとAZ / SLAの読み方 / 「クラウドなら安全」の誤解

### 5-2 Microsoft 365管理

- **学習ゴール**: M365管理センターでユーザー・ライセンス・Exchange・Teams・SharePointの日常管理ができる
- **教材構成**
  - 動画「M365管理センターツアー」(15分)
  - ラボ「M365管理演習」(150分・M365開発者/試用テナント使用): ユーザー作成とライセンス付与 → 共有メールボックス作成 → Teamsチーム設計 → SharePoint権限 → 監査ログ検索
  - HTML「Entra ID入門」: オンプレADとの違い、ハイブリッド構成(Entra Connect)概要
  - クイズ: 10問
- **内容トピック**: ライセンス体系(E3/E5/Business)と費用感 / Exchange Onlineの運用(転送・配布グループ・迷惑メール)/ OneDrive/SharePointの共有制御 / 条件付きアクセス(概要)

### 5-3 Google Workspace管理

- **学習ゴール**: Google管理コンソールで組織部門・グループ・共有設定の管理ができる
- **教材構成**
  - ラボ「Google Workspace管理演習」(120分・無料試用): 組織部門設計 → ユーザー/グループ → ドライブ共有ポリシー → 端末管理 → 監査ログ
  - HTML「M365 vs Google Workspace 管理項目対応表」(両刀使いになるための翻訳表)
- **内容トピック**: 組織部門とポリシー継承 / 共有ドライブ設計 / 外部共有制御 / データエクスポート(退職時)

### 5-4 ID管理とSSO(SAML/OIDC/MFA)

- **学習ゴール**
  1. SSOの価値(利便性とセキュリティの両立)を非IT役員に説明できる
  2. SAML認証フローの登場人物(IdP/SP)と流れを図で説明できる
  3. MFAの方式別強度(SMS<アプリ<FIDO2)を説明できる
- **教材構成**
  - 動画「パスワード100個問題とSSO」(12分): SSO導入前後の社員体験をドラマ仕立てで
  - HTML「SAML/OIDCフロー図解」: シーケンス図を段階アニメーション表示
  - クイズ: 12問(「このエラーはIdPとSPどちらを疑う?」等の実務問題)
- **内容トピック**: IdP/SP / SAMLアサーション(深入りせず流れ重視)/ OIDCとOAuthの違い(概要)/ プロビジョニング(SCIM)/ MFA方式比較とMFA疲労攻撃 / パスキー

### 5-5 AWS基礎(VPC/EC2/S3/IAM)

- **学習ゴール**: 無料枠でVPC+EC2+S3の最小構成を構築でき、IAMで最小権限のユーザーを作れる
- **教材構成**
  - 動画「AWSの地図」(12分): 主要サービスをオンプレ用語に翻訳(EC2=仮想サーバー、S3=倉庫、VPC=社内LAN)
  - ラボ「AWS最小構成構築」(180分): アカウント開設(**予算アラート設定を最初に必須化**)→ IAMユーザー+MFA → VPC/サブネット → EC2起動(SSH接続、Part 3の知識を接続)→ S3バケット → 後片付け(課金防止の削除手順を厳格に)
  - クイズ: 10問
- **内容トピック**: ルートユーザーを使わない / セキュリティグループ / 従量課金と見積もり / S3公開事故の防止 / Azureとの用語対応表

### 5-6 SaaSの選定と管理

- **学習ゴール**: SaaS導入の評価軸(機能・セキュリティ・費用・サポート)で比較表を作り、導入後の管理プロセスを設計できる
- **教材構成**
  - ケーススタディ(分岐型・2シナリオ):
    1. 「営業部が勝手にSaaS契約していた(シャドーIT発覚)」: 頭ごなし禁止か、公認化か、の対応分岐
    2. 「経費精算SaaSを3製品から選定せよ」: RFP項目の取捨選択→選定理由書の作成
  - HTML「SaaS管理台帳テンプレート」+「セキュリティチェックシート(委託先評価)」
- **内容トピック**: シャドーITの実態と対策 / アカウントライフサイクル管理 / SaaSコスト最適化(未使用ライセンス回収)/ 解約時のデータ引き上げ

### 5-7 Part 5 修了テスト
