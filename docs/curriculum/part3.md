# Part 3 Linuxサーバー入門〜実践(全13章 / 約40時間 / レベル2)★最重量パート

## パート概要

本コースの中核。コマンド暗記で終わらせず、**viでの設定編集 → nginxの設定読解 →
自力での環境構築 → ログを読んだトラブルシュート** まで到達させる。
全章にハンズオンラボを置き、「読んだだけのLinux」を絶対に作らない。

**到達目標**: Ubuntu Server を自力で構築し、nginx の設定ファイルを読解・変更・検証でき、
ログから障害原因を特定できる。

**ラボ環境**: VirtualBox + Ubuntu Server 24.04 LTS(3-2章で構築)。
Windowsユーザー向けに WSL2 代替手順も全ラボに併記する。

## 各章詳細

### 3-1 Linuxとは・ディストリビューションの選び方

- **学習ゴール**: Linuxが社内のどこで動いているかを挙げられ、Ubuntu/RHEL系の系統と選定観点を説明できる
- **教材構成**

  - HTML「ディストリビューション系統図」: Debian系/RHEL系、LTSとサポート期間、商用サポートの考え方

### 3-2 学習環境構築(VirtualBox+Ubuntu Server)

- **学習ゴール**: 仮想マシンに Ubuntu Server をインストールし、スナップショットで「壊しても戻せる」環境を持つ

  - ラボ「マイサーバー構築」(120分):
    1. VirtualBoxインストール → 2. ISO取得と検証 → 3. VM作成(CPU/メモリ/ディスク割り当ての意味を解説)

    → 5. ネットワーク設定(NAT+ホストオンリーの2枚構成)→ 6. スナップショット取得

  - 動画「仮想化ってなに?」(8分): ハイパーバイザーの概念図解
- **つまずき対策**: VT-x無効・Hyper-V競合・ISO署名検証エラーなどFAQを厚めに用意

### 3-3 基本コマンドI(ファイル操作・パイプ・リダイレクト)

- **学習ゴール**
  1. ディレクトリ移動・ファイル操作(`ls` `cd` `pwd` `cp` `mv` `rm` `mkdir` `cat` `less`)を迷わず使える
  2. 絶対パスと相対パスを使い分けられる
  3. パイプ(`|`)とリダイレクト(`>` `>>` `2>`)でコマンドを組み合わせられる
- **教材構成**
  - ラボ「ファイル操作100本ノック(前半50本)」(90分): お題形式。「/tmp配下に部署別ディレクトリを作り…」と業務風の指示で操作
  - ドリル「コマンド当て道場」: 「画面に収まらない長いファイルを見るには?」→ 反射で `less`
  - HTML「コマンドチートシート(印刷用A4・1枚)」
- **内容トピック**: シェルの仕組み(プロンプト・引数・オプション) / タブ補完と履歴 / ワイルドカード / `rm -rf` 事故防止の作法(まず `ls` で当てる)

### 3-4 基本コマンドII(検索・圧縮・プロセス)

- **学習ゴール**
  1. `grep` `find` でファイル・文字列を探せる(業務最頻出)
  2. `tar` `gzip` `zip` でアーカイブを扱える
  3. `ps` `top` `kill` でプロセスを確認・制御できる
- **教材構成**
  - ラボ「100本ノック(後半50本)」(90分): 「設定ファイル群から特定IPを含む行を探す」「先週のログだけ固めて退避」等の実務シナリオ

  - クイズ: 10問
- **内容トピック**: `grep -r -n -i -v` / `find -name -mtime -size` / パイプ合わせ技(`ps aux | grep nginx`)/ シグナル(TERMとKILLの違い)/ `df` `du` `free` でリソース確認

### 3-5 viエディタ完全入門 ★重点章

- **学習ゴール**
  1. モードの概念(ノーマル/挿入/コマンドライン/ビジュアル)を理解し、**「viに閉じ込められる」状態から卒業**する
  2. 移動・編集・検索置換を実務速度で行える
  3. サーバー上の設定ファイルを安全に編集する作法(バックアップ→編集→検証)を身につける
- **教材構成**
  - 動画①「viはなぜ難しい?モードを理解すれば怖くない」(12分): モード遷移図を中心に。「まず`Esc`、困ったら`:q!`」を合言葉化
  - 動画②「vi実演:nginx設定を編集してみよう」(10分): 実際の編集風景を打鍵表示付きで
  - **ドリル「viキー道場」(3レベル)**: 画面に指示が出て正しいキーを打つ反射訓練
    - Lv1 生存編: `i` `Esc` `:w` `:q` `:q!` `:wq`
    - Lv2 移動編: `h j k l` `w` `b` `0` `$` `gg` `G` `:数字` `Ctrl-f/b`
    - Lv3 編集編: `x` `dd` `yy` `p` `u` `Ctrl-r` `/検索` `n` `:%s/old/new/g` `.`(リピート)
  - ラボ「vi実務演習」(60分): サンプル設定ファイルを配布し、指示書どおりに編集 → diffで自己採点
  - HTML「viチートシート(1枚)」+「vimとの違い・最低限の`~/.vimrc`(行番号・シンタックス)」
- **内容トピック**: モード遷移 / カーソル移動 / 削除・コピー・貼り付け(レジスタの初歩)/ 検索・置換 / アンドゥ / 複数行操作(ビジュアルモード)/ `sudoで開き忘れた時の対処` / nanoとの使い分け(viを主、nanoは緊急避難と位置づけ)

### 3-6 ユーザー・グループとパーミッション

- **学習ゴール**
  1. `rwx` 表記と数値表記(755/644/600)を相互変換できる
  2. ユーザー・グループを設計してファイル共有権限を組める
  3. `sudo` の仕組みと危険性を理解する
- **教材構成**
  - ラボ「権限設計演習」(75分): 「経理チーム共有ディレクトリ(経理は読み書き、他部署は閲覧のみ、外部は不可)」を `useradd`/`groupadd`/`chmod`/`chown` で構築
  - クイズ: 12問(「このlsの出力で、ユーザーXは書き込める?」形式の読解問題多め)
- **内容トピック**: `ls -l` の読み方 / chmod のシンボリック・数値両記法 / umask / setuid・sticky bit(概要)/ sudoers と `sudo` の作法 / root直ログイン禁止の理由

### 3-7 パッケージ管理とsystemd

- **学習ゴール**
  1. `apt` でパッケージの検索・導入・更新・削除ができる
  2. `systemctl` でサービスの起動・停止・自動起動設定ができる
  3. `journalctl` でサービスのログを確認できる
- **教材構成**
  - ラボ「サービス管理演習」(75分): `apt update/upgrade` の違いを確認 → nginxを導入 → `systemctl status/start/stop/enable` → ユニットファイルを覗く → `journalctl -u nginx` でログ追跡
  - HTML「systemd入門」: ユニットの種類、`enable`と`start`の違いの図解
  - クイズ: 10問
- **内容トピック**: リポジトリの概念 / `apt` と `dpkg` / セキュリティアップデート運用 / systemdユニット / 起動失敗時の調査手順(`status`→`journalctl -xe`)

### 3-8 シェルスクリプト入門とcron

- **教材構成**
  - ラボ「はじめての運用自動化」(120分): ①ディレクトリを日付付きtarで退避するバックアップスクリプト → ②古い世代を削除するローテーション → ③cron登録 → ④実行ログ出力
  - HTML「シェルスクリプト文法ミニマム」: 情シス運用に必要な範囲だけ(変数・if・for・引数・終了コード)に絞る
- **内容トピック**: シバン / 変数とクォート / `if [ ]` / `for` / 終了コードと `&&` `||` / crontab の書式(5フィールドの読み書き)/ 事故防止(まず `echo` でドライラン)

### 3-9 SSHとファイル転送

- **学習ゴール**
  1. 公開鍵認証の仕組みを図で説明でき、鍵ペアを作って設定できる
  2. `scp`/`rsync` でファイルを安全に転送できる
  3. `sshd_config` の主要設定(ポート・rootログイン禁止・パスワード認証禁止)を読める
- **教材構成**
  - ラボ「SSH要塞化」(90分): ホストPC→VMへ鍵認証設定 → パスワード認証無効化(★viで `sshd_config` を編集し、**設定反映前に別セッションを維持する作法**を体験)→ `scp`/`rsync` 転送 → `~/.ssh/config` で接続簡略化
  - クイズ: 10問
- **内容トピック**: 公開鍵と秘密鍵 / known_hosts / パーミッション要件(700/600)/ ポートフォワード(概要)/ 締め出し事故の防ぎ方

### 3-10 nginx入門:インストールと設定の読み方 ★重点章

- **学習ゴール**
  1. `nginx.conf` の構造(ディレクティブ・ブロック・include)を読解できる
  2. `server` / `location` ブロックのマッチングルールを理解し、バーチャルホストを設定できる
  3. リバースプロキシ設定を読み書きできる
  4. 設定変更の安全手順(`nginx -t` → reload)を習慣化する
- **教材構成**
  - 動画①「nginxってなに?Webサーバーとリバースプロキシ」(12分): 「受付窓口」比喩でプロキシを図解
  - 動画②「nginx.conf 読解ツアー」(15分): デフォルト設定を上から1行ずつ読む実演
  - ラボ「nginx設定演習」(150分):
    1. デフォルトページ確認 → 設定ファイルの全体構造を把握(`nginx.conf` と `sites-available/enabled` の関係)
    2. 静的サイトのバーチャルホスト追加(`server_name` / `root` / `index`)
    3. `location` マッチ実験(前方一致・完全一致 `=`・正規表現 `~` の優先順位を実際に当てて確認)
    4. リバースプロキシ(`proxy_pass` でローカルのテスト用アプリへ転送、`proxy_set_header` の意味)
    5. 自己署名証明書でHTTPS化(`listen 443 ssl`、リダイレクト設定)
    6. 各ステップで **`nginx -t` → `systemctl reload nginx` → curlで検証** を徹底
  - HTML「nginx設定リファレンス」: 実務で見る頻出ディレクティブ30個を1行注釈付きで
  - クイズ: 12問(実際の設定断片を読ませて「このURLはどのlocationに当たる?」を問う読解問題中心)
- **内容トピック**: ディレクティブとコンテキスト(main/http/server/location)/ include構造 / listen と server_name / locationマッチ優先順位 / try_files / proxy_pass(末尾スラッシュの罠)/ アクセスログ・エラーログの場所 / reload と restart の違い

### 3-12 ログの読み方とトラブルシュート

- **学習ゴール**: `/var/log` 配下と `journalctl` から障害の原因を特定する型を身につける
- **教材構成**
  - ケーススタディ(分岐型・3シナリオ): 壊れた状態のVMスナップショット(または再現手順)を配布
    1. 「Webサイトが502 Bad Gateway」(→ PHP-FPM停止をログから発見)
    2. 「ディスクフルでサービス停止」(→ `df`→`du`→ログ肥大を発見しローテーション設定)
    3. 「再起動したらnginxが上がらない」(→ 設定ミスを `nginx -t` と journalctl で特定)
  - ドリル「ログ読解道場」: 実ログ断片から「時刻・レベル・原因」を読み取る反復問題
- **内容トピック**: syslogとjournald / ログレベル / アクセスログのフォーマット読解(ステータスコード別の意味)/ logrotate / 「エラーメッセージで検索する」技術
